# AI 共同開発の記録

本プロジェクトは **Claude (Anthropic)** との対話駆動で開発した。
人間と AI の分業を可視化する記録。

## 分業モデル

| 担当 | 人間 (HIROHISA-S) | AI (Claude) |
|---|---|---|
| 仕様判断 | ★★★★★ | ★ (提案のみ) |
| UX 設計判断 | ★★★★★ | ★★ (選択肢提示) |
| アーキテクチャ設計 | ★★★ | ★★★★ |
| 実装 (コード) | ★ | ★★★★★ |
| デバッグ | ★★★ | ★★★★ |
| 信号処理ロジック | ★★★ | ★★★★ |
| ドキュメント | ★★ | ★★★★ |

## 工程ハイライト

### Phase 0: 基盤
- Muse S → Mind Monitor → OSC 受信パイプライン構築
- Raw EEG / Band Power / Spectrogram / HR 可視化
- Claude が pyqtgraph 構成を提案、人間がレイアウト承認

### Phase 1: EQ 制御
- pedalboard で 3-band → 6-band に拡張
- 「Spotify アプリと差別化されない」と人間が判断 → AI が代替案 5つ提示
- 案 4 (Emotional Seascape) を人間が選択

### Phase 1.5: Seascape
- 当初手描き procedural 描画 → 「リアルさが足りない」と人間判断
- AI が Veo プロンプト (3 シーン) を生成
- 人間が Veo で動画生成 → AI が cv2 ベースのプレイヤー実装
- DirectShow エラー検出 → AI が OpenCV 代替を提案

### Phase 2: Underwater 統合 + Watch サブビュー化
- 人間がシーン構成を再判断 (Storm → Underwater)
- AI が新プロンプト & コード差し替え
- 3 動画 (low/mid/high) を HR ヒステリシスで切替
- City サブビュー追加 (HR シンクで動的演出)

### Phase 3: UI 大幅刷新 (3 モード化)
- Studio / Listen / Watch の 3 モードに分離
- AI が独立した paintEvent カスタムウィジェットを多数提案・実装
  - `_ParticleEEG` (流れるパーティクル波形)
  - `_BandSphere` (ネオンリングゲージ)
  - `_RussellPad` (Russell 円)
  - `_MandalaOverlay` (神経網オーブ)
  - `_MatrixRain` / `_TronGrid` (Watch 背景)
  - `_RibbonBar` (流体感情リボン)
  - `_InstrumentCircle` (Listen 楽器サークル)
- 人間が mockup 提示 → AI が「これは作れる/作れない」仕分け → 実装
- パフォーマンス問題発生時 (h264 デコード, パーティクル数) → AI が即座に最適化

### Phase 4: アセット統合
- 人間が Veo / Imagen で画像・動画生成
- AI が PIL でクロップ + 配置 + コード組み込み
- 楽器テクスチャ 6種 / 回路パターン / 都市夜景 すべて統合

### Phase 5: 微 UX 強化
- スプラッシュスクリーン

- モード切替スライドアニメ
- 品質ドット呼吸
- オーディオレベルメータ

## 学んだこと

### AI を使う上で有効だった習慣

1. **コンテキストを毎回与え直す** — 長いセッションでも要点 (今の感情マッピング, ファイル構成) を渡す
2. **「なぜ」を必ず聞く** — 実装案だけでなく、トレードオフを言わせる
3. **正直な評価を求める** — 「Muse の精度どう?」「これって他で代替できる?」など
4. **小さく動かす** — 1ファイルずつ動作確認、丸ごと書き直し回避

### AI が苦手だったこと

- **長期記憶** — セッション跨ぎでは設計判断が薄れる → ドキュメント化で補う
- **ハードウェア依存の判断** — Bluetooth latency など実機テストが必要
- **デザインの趣味判断** — 「もっとカッコよく」みたいな曖昧指示には弱い

## 結論

- **コード ~6000 行** (主要 6 ファイル + ドキュメント + アセット)
- AI なしでこのスピード + 質は実現不可能だったと判断
- 一方で**判断は全て人間が下した** — AI は実装担当の優秀な同僚

## 数字で見る AI 活用

| 指標 | 値 |
|---|---|
| Python LoC | ~6000 (`realtime_monitor.py` 6200+, `sea_widget.py` 1000+, 他) |
| カスタムウィジェット (paintEvent) | 15+ (Particle EEG, Band Sphere, Russell Pad, Mandala, Matrix Rain, Tron Grid, Ribbon Bar, Demo Explainer 等) |
| AI 生成アセット | 動画 (Veo) + 画像 (Imagen) |
| AI に書かせたコード割合 | ~95% (人間は構造判断と仕様判断) |
| AI が踏んだトラブルを人間が指摘 | h264 デコード問題 / pip 経路 / VB-CABLE デバイス選定 etc. |

## AI 共同開発のテンプレ的なやり方

1. **人間が mockup や望む状態を見せる** (画像 / 既存実装の挙動)
2. **AI に「できる/できないを先に正直に」と聞く**
3. **AI が仕分け** → 実現可能なものから着手
4. **動作確認 → 微調整** のループ
5. 詰まったら**人間が「別の方法は?」**と問い返す

このバランスが、これからの **「人と AI の協調」** 開発の一つの形だと考えている。
